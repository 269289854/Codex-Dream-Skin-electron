# Windows 使用指南

## 环境要求

- Windows 10 或 Windows 11 x64。
- Microsoft Store 安装的官方 Codex 桌面应用。
- 安装版不要求单独安装 Node.js；开发构建需要 Node.js 22 或更新版本。

## 制作主题

左侧用于创建、复制、删除和切换主题。内置的系统主题会固定标记为“系统主题”，可以修改、复制和导出，但不能删除；新建、复制及分享导入的主题均标记为“自定义主题”，可以正常删除。内置系统预设仅在首次安装或系统主题丢失时创建，软件升级不会覆盖用户已经修改的系统主题；在系统主题中点击“恢复默认”会重新载入完整的内置主视觉、拍立得和雨落特效，点击“保存主题”后正式写入。中间是按照 Codex 当前布局制作的实时预览，右侧用于编辑当前主题。

在“视觉设计”中选择主视觉媒体，并使用缩放、水平位置和垂直位置调整背景。选择媒体后可单独开启“水平翻转”或“垂直翻转”，翻转只影响图片内容，不会镜像首页文字和操作卡片。支持 PNG、WebP、JPEG、SVG、原生 GIF、MP4 和 WebM；SVG 会先校验并转换为 PNG。GIF 最大 30 MB；视频最长边不能超过 4096px，视频不设固定文件大小上限，但导入、复制和分享时必须保留至少 10 GB 或磁盘 15% 的可用空间。媒体会复制到 `%LOCALAPPDATA%\CodexDreamSkinStudio`，删除原始文件不会影响已经保存的主题。

视频的自动播放、循环、声音和音量在预览配置页设置。默认自动播放、循环和静音；主视觉与拍立得最多一个位置可以开启声音。浏览器阻止自动播放时会保留画面并显示播放按钮。GIF 遵循文件自身的动画和循环行为，不显示视频播放或声音控件。

“视觉设计”顶部的“整个窗口背景”控制 Codex 完整应用视口，包括顶部工具栏、左侧栏和右侧主区域。背景固定在窗口中，不会随首页/会话切换或内容滚动。颜色模式可以使用纯色、线性渐变或径向渐变；线性渐变支持角度，径向渐变支持中心位置，两种渐变都支持 2–8 个色标。媒体模式支持 PNG、WebP、JPEG、原生 GIF、MP4 和 WebM，并可调整透明度、焦点、缩放及水平/垂直翻转。窗口背景视频始终静音、自动播放、循环并使用内联播放。

启用整个窗口背景后，Studio 预览和实际 Codex 会自动把顶部工具栏、侧栏、主区域、品牌栏等结构表面设为透明，让窗口背景透出；卡片、消息、输入框、边框和装饰仍使用各自的主题样式。关闭整个窗口背景后，这些结构表面恢复原有配置，不会丢失用户设置。

窗口背景可以添加最多 8 层遮罩。图层列表从上到下按“前景到背景”排列，“上移”会使遮罩更靠前；重叠区域按各层透明度正常合成。每层可以独立显示或隐藏、复制、删除和排序，并设置纯色/渐变、透明度、全屏/椭圆/圆角矩形、位置、宽高、边缘柔化和圆角。删除全部遮罩即为“无遮罩”。窗口背景遮罩位于应用内容下方，不会挡住文字、按钮或窗口控制。

窗口背景与“对话区域背景”独立保存。会话背景启用时显示在窗口背景上方；关闭会话背景时，对话区域会透出整个窗口背景。切换任一背景的颜色、图片、GIF 或视频不会清除另一套背景和遮罩设置。

切换到中间预览的会话页面后，可以直接点击对话区域打开“对话区域背景”设置。背景支持纯色、PNG/WebP/JPEG 图片、原生 GIF、MP4 和 WebM，并可单独调整背景透明度、媒体焦点和缩放。背景遮罩支持纯色、线性渐变和径向渐变；线性渐变可调整角度，径向渐变可调整中心，渐变可以添加 2–8 个色标并分别设置颜色与位置。

遮罩形状可以选择全屏、椭圆或圆角矩形。椭圆和圆角矩形只在形状内部着色，形状外保留原背景；可以调整水平/垂直位置、宽度、高度和边缘柔化，圆角矩形还可以调整圆角。切换背景媒体类型不会清除遮罩配置。保存主题并重新注入后，Studio 预览与 Codex 会使用相同的渐变和几何参数。

“首页文案”可以配置首页标题和副标题。标题必须且只能保留一个 `{project}`，注入后该占位符会显示为 Codex 原生项目选择器；副标题留空时不显示。中间预览使用固定的 `1280 × 820` Codex 画布并等比缩放，标题、四张快捷卡片、项目栏、输入框和拍立得与实际注入共用同一套布局样式。

拍立得使用单独的媒体设置。拍立得也有独立的水平/垂直翻转开关；翻转只作用于裁剪区域中的图片、GIF 或视频，相框、裁剪围栏、阴影、图钉、旋转和拖动位置保持不变：

1. 点击“选择拍立得媒体”。
2. 点击指针按钮后，依次选择拍立得的四个角点。
3. 拖动编号角点进行修正，围栏不能自交或超出图片。
4. 在中间预览中直接拖动拍立得，并在右侧调整宽度、旋转和隐藏阈值。
5. 点击“保存主题”。

四点坐标相对原图归一化保存，因此更换显示器或窗口尺寸不会改变原图中的裁切位置。

Studio 中保存的拍立得位置是每次注入的初始位置。主题注入到 Codex 后，用户可以直接在 Codex 页面内拖动拍立得；这个临时位置只在当前页面生命周期内生效，不会写回主题，也不会同步到 Studio 预览。重新注入或重启 Codex 后会恢复到 Studio 中保存的位置。

## 分享主题

左侧主题操作区的下载按钮会把当前预览中的设计（包括尚未保存的修改）导出为单个 `.cdstheme` 文件。文件选择器中的上传按钮可以导入分享文件，也可以直接把 `.cdstheme` 拖到 Studio 窗口的释放区域。导入成功后会创建一个新的本地主题并自动激活、打开；不会覆盖已有主题，同名主题也可以并存。

分享文件是自包含 ZIP64 包，包含主题配置、实际引用的图片、GIF、视频、定制图标和字体，因此接收方不需要原始素材文件。导入前应用会校验文件清单、路径、主题 schema、素材大小、媒体头部和 SHA-256；不接受脚本或可执行文件。分享导入按流式方式处理并受可用磁盘空间保护，条目最多 128 个，单个图片/GIF 最多 30 MB、字体最多 12 MB。校验或复制失败时会清理临时数据，当前主题保持不变。

视频在 Codex 页面内通过已验证的主题文件绑定生成 `blob:` 地址，Studio 和渲染进程不会获得真实磁盘路径，也不会关闭 Codex CSP。主题运行期间关闭 Studio 窗口会驻留系统托盘，停止主题后释放媒体绑定。

## 图标和颜色

“图标样式”提供品牌、主卡片、副卡片、输入框、项目、装饰和拍立得图钉槽位。每个槽位可选择内置 Lucide 图标，或导入 PNG、WebP、JPG、SVG。SVG 会先检查外部引用和脚本，再转换为 PNG。

颜色设置包含背景、正文、强调、粉色、淡紫、边框、成功和危险状态色。保存后重新注入即可应用到运行中的 Codex。

## 启动和恢复

“运行设置”包含以下操作：

- “检测 Codex”：只读检查 Store 安装、版本和运行状态。
- “安装配置”：备份并设置 Codex 的浅色基础主题。
- “启动并应用”：必要时确认重启 Codex，随后开启仅监听回环地址的 CDP 端口并注入主题。
- “重新注入”：保存主题后刷新运行中的 Codex 页面。
- “验证主题”：检查所有 Codex 页面是否存在主题标记和样式。
- “停止注入”：移除当前页面主题，但不恢复基础配置。
- “恢复并重启 Codex”：恢复安装前的配置，归档备份并以普通模式重启 Codex。

主题活动时关闭 Studio 窗口只会隐藏到系统托盘。托盘中的“恢复 Codex 并退出”会先恢复再退出。

## 数据和安全

应用数据位于 `%LOCALAPPDATA%\CodexDreamSkinStudio`：

- `themes/<id>/theme.json`：主题配置。
- `themes/<id>/assets/`：导入素材。
- `backups/`：Codex 配置备份和恢复归档。
- `runtime/`：当前注入载荷和会话状态。

CDP 只接受 `127.0.0.1`、`localhost` 或 `::1`，并校验端口、浏览器 ID、页面 ID、`app://` 页面和 Store AppX 进程归属。渲染进程不能直接访问文件系统、PowerShell 或任意 CDP 地址。

## 故障排查

- 检测不到 Codex：确认安装的是 Microsoft Store 官方版本，然后重启 Studio。
- 端口被占用：关闭占用 `9335` 的程序后重试。
- 主题没有出现：打开“运行设置”执行“重新注入”和“验证主题”。
- Studio 意外退出：重新打开后会尝试恢复活动会话；也可以执行“恢复并重启 Codex”。
- 配置错误：不要手工删除 `backups/`，先使用 GUI 恢复。
