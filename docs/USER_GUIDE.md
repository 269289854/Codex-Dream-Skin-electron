# Windows 使用指南

## 环境要求

- Windows 10 或 Windows 11 x64。
- Microsoft Store 安装的官方 Codex 桌面应用。
- 安装版不要求单独安装 Node.js；开发构建需要 Node.js 22 或更新版本。

## 制作主题

左侧用于创建、复制、删除和切换主题。内置的系统主题会固定标记为“系统主题”，可以修改、复制和导出，但不能删除；新建、复制及分享导入的主题均标记为“自定义主题”，可以正常删除。中间是按照 Codex 当前布局制作的实时预览，右侧用于编辑当前主题。

在“视觉设计”中选择主视觉图片，并使用缩放、水平位置和垂直位置调整背景。主题图片会复制到 `%LOCALAPPDATA%\CodexDreamSkinStudio`，删除原始文件不会影响已经保存的主题。

“首页文案”可以配置首页标题和副标题。标题必须且只能保留一个 `{project}`，注入后该占位符会显示为 Codex 原生项目选择器；副标题留空时不显示。中间预览使用固定的 `1280 × 820` Codex 画布并等比缩放，标题、四张快捷卡片、项目栏、输入框和拍立得与实际注入共用同一套布局样式。

拍立得使用单独的图片设置：

1. 点击“选择拍立得原图”。
2. 点击指针按钮后，依次选择拍立得的四个角点。
3. 拖动编号角点进行修正，围栏不能自交或超出图片。
4. 在中间预览中直接拖动拍立得，并在右侧调整宽度、旋转和隐藏阈值。
5. 点击“保存主题”。

四点坐标相对原图归一化保存，因此更换显示器或窗口尺寸不会改变原图中的裁切位置。

## 分享主题

左侧主题操作区的下载按钮会把当前预览中的设计（包括尚未保存的修改）导出为单个 `.cdstheme` 文件。文件选择器中的上传按钮可以导入分享文件，也可以直接把 `.cdstheme` 拖到 Studio 窗口的释放区域。导入成功后会创建一个新的本地主题并自动激活、打开；不会覆盖已有主题，同名主题也可以并存。

分享文件是自包含 ZIP 包，包含主题配置、实际引用的图片、定制图标和字体，因此接收方不需要原始素材文件。导入前应用会校验文件清单、路径、主题 schema、素材大小、图片内容、字体头部和 SHA-256；不接受脚本或可执行文件。单个分享包最多 500 MB 压缩数据、1 GB 解压数据和 128 个条目，单个图片最多 30 MB、字体最多 12 MB。校验或复制失败时会清理临时数据，当前主题保持不变。

## 图标和颜色

“图标样式”提供品牌、主卡片、副卡片、输入框、项目、装饰和拍立得图钉槽位。每个槽位可选择内置 Lucide 图标，或导入 PNG、WebP、JPG、SVG。SVG 会先检查外部引用和脚本，再转换为 PNG。

颜色设置包含背景、正文、强调、粉色、淡紫、边框、成功和危险状态色。保存后重新注入即可应用到运行中的 Codex。

## 启动和恢复

“运行设置”包含以下操作：

- “检测 Codex”：只读检查 Store 安装、版本和运行状态。
- “安装配置”：备份并设置 Codex 的浅色基础主题。
- “启动并应用”：必要时确认重启 Codex，随后开启仅监听回环地址的 CDP 端口并注入主题。
- “重新注入”：保存主题后刷新运行中的 Codex 页面。
- “验证主题”：检查所有 Codex 页面是否存在主题标记和样式。
- “停止注入”：移除当前页面主题，但不恢复基础配置。
- “恢复并重启 Codex”：恢复安装前的配置，归档备份并以普通模式重启 Codex。

主题活动时关闭 Studio 窗口只会隐藏到系统托盘。托盘中的“恢复 Codex 并退出”会先恢复再退出。

## 数据和安全

应用数据位于 `%LOCALAPPDATA%\CodexDreamSkinStudio`：

- `themes/<id>/theme.json`：主题配置。
- `themes/<id>/assets/`：导入素材。
- `backups/`：Codex 配置备份和恢复归档。
- `runtime/`：当前注入载荷和会话状态。

CDP 只接受 `127.0.0.1`、`localhost` 或 `::1`，并校验端口、浏览器 ID、页面 ID、`app://` 页面和 Store AppX 进程归属。渲染进程不能直接访问文件系统、PowerShell 或任意 CDP 地址。

## 故障排查

- 检测不到 Codex：确认安装的是 Microsoft Store 官方版本，然后重启 Studio。
- 端口被占用：关闭占用 `9335` 的程序后重试。
- 主题没有出现：打开“运行设置”执行“重新注入”和“验证主题”。
- Studio 意外退出：重新打开后会尝试恢复活动会话；也可以执行“恢复并重启 Codex”。
- 配置错误：不要手工删除 `backups/`，先使用 GUI 恢复。
