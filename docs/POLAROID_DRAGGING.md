# 拍立得拖拽与保存

## 行为

实际 Codex 首页中的整张拍立得支持鼠标、触控笔和触摸拖拽。按下后使用 Pointer Capture 持续接收移动事件，释放或取消时结束拖动。

- 拖动只改变拍立得位置 `x/y`，不会改变宽度、旋转角度、围栏或隐藏阈值。
- 位置被限制在当前主题内容区域内，避免拍立得被完全移出可见区域。
- 拖动期间立即使用内联位置更新画面，不等待后台保存。
- 拖动结束后自动保存到当前主题；刷新、重启和重新注入后继续使用保存的位置。
- 保存成功后重新同步所有 Codex 页面，并将位置合并到 Studio 当前草稿。

## 数据流

1. 注入脚本在拖动结束时生成一次性 `{ themeId, x, y }` 消息。
2. CdpWatcher 使用现有后台轮询读取并清除消息，拒绝错误主题 ID、非数字和越界坐标。
3. CodexService 串行更新当前主题，只写入拍立得的 `x/y`。
4. 主进程重建运行时 payload 并重新注入已连接页面。
5. 内部 IPC 通知 Studio；Studio 仅合并位置，保留其他尚未保存的编辑内容。

保存失败时，当前页面仍保留本次临时位置，Studio 运行状态显示失败信息；主题文件继续保持最后一次成功保存的有效内容。

## 隐私约束

位置消息只包含主题 UUID 和标准化坐标，不读取或传输项目、任务、账号、团队、邮箱或路径信息。实现、测试和验收继续遵守 [隐私与示例数据约束](PRIVACY.md)。
