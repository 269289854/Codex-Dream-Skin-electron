# 首页卡片图标预览对齐

## 问题与修复

实际 Codex 的默认首页功能卡分别使用操作自身的 `</>`、`+`、`✓` 和 `✦` 字形，而 Studio 曾使用主卡片和副卡片主题槽位中的通用 Lucide SVG，导致四个小图标的预览外观与实际页面不同。

Studio 首页卡片预览和实际 Codex 注入现在使用相同的回退规则：默认主卡片和副卡片槽位显示操作自身的四个字形；导入的自定义图片仍然优先显示；用户改选其他内置图标时使用 `src/shared/icon-glyphs.ts` 中的共享映射。装饰统一使用 `♥`。

本次不修改主题文件版本、配置结构、IPC/API、卡片文案或布局尺寸。

## 回归与隐私约束

- 预览测试必须验证卡片使用实际注入渲染模式及共享字形映射。
- DOM 测试必须验证实际注入后的四张卡片图标和装饰字形。
- 运行时 payload 测试必须验证共享映射被传入注入页面，注入脚本不得另存一份重复映射。
- 实现、测试、文档和验收材料不得包含真实项目、任务、账号、团队或本机路径，并继续遵守 [隐私与示例数据约束](PRIVACY.md)。
- 验收只使用后台测试和用户提供的参考图，不控制用户前台窗口。

## 验收方式

运行预览布局测试、注入 DOM 测试、完整测试、类型检查、生产构建和 `git diff --check`。视觉重点检查四张卡片依次显示 `</>`、`+`、`✓`、`✦`，底部均为实心心形，且图标尺寸、圆形底座和卡片布局不发生变化。
