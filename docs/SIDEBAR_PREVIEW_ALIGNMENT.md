# 侧边栏预览对齐说明

## 改造结果

Studio 的 Codex 预览已从旧版“双栏侧边栏”改为当前 Codex 风格的单栏侧边栏。侧边栏总宽度保持为 270px，因此右侧首页预览的起始位置和主题参数计算不受影响。

预览现在包含：

- Codex 模式标题、音乐符号和搜索按钮。
- 带 Lucide 图标的新建任务、拉取请求、站点、已安排和插件导航。
- 独立的项目滚动区、当前项目选中态、任务行和无任务状态。
- 与滚动区分离的固定团队底栏。
- 位于主内容品牌区的可配置 `branding` 图标。

实际 Codex 注入逻辑、主题配置格式和 IPC 接口均未修改。

## 示例数据与隐私

侧边栏使用固定的虚构示例快照，不读取运行中 Codex 的项目、任务、账号或团队数据。当前公开仓库名可以用于当前项目，其余项目、任务和团队标识必须是虚构内容。

所有后续改动必须遵守 [隐私与示例数据约束](PRIVACY.md)：真实项目名、任务标题、账号名、团队名、邮箱、本机路径和未脱敏截图不得进入源码、测试、文档、日志或提交记录。`tests/preview-layout.test.ts` 使用完整允许列表锁定当前示例项目和虚构团队标识。

## 验证结果

2026-07-17 完成以下后台验证：

- `npm run typecheck`：通过。
- `npm run build`：主进程、preload 和 renderer 生产构建通过。
- `git diff --check`：通过。
- 敏感名称扫描：截图中出现过的本机项目、任务和账号关键词在仓库内容中无匹配。

## 视觉验收

视觉检查通过只读抓取现有 Studio 窗口完成，没有激活窗口、移动鼠标、发送键盘输入或控制其他应用。检查确认：

- 旧图标轨道已消失，侧边栏为完整的 270px 单栏。
- 导航、项目名称、任务层级和滚动条没有重叠或溢出。
- 项目滚动区不会遮挡固定团队底栏。
- 当前项目选中态使用主题青粉色，并保持长名称省略。
- 主内容品牌图标、首页卡片、拍立得和输入区没有因侧边栏改造发生位移回归。
- 验收画面仅包含固定虚构示例数据；临时截图未写入仓库。
