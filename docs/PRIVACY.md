# 隐私与示例数据约束

本仓库是公开项目。源码、测试、文档、日志和可提交的图片中不得出现任何开发者本机或真实 Codex 环境的数据。

## 强制规则

- 禁止提交真实项目名、仓库名、任务标题、分支名、工作区名称或未公开的业务描述。
- 禁止提交真实账号名、团队名、邮箱、头像标识、组织名称、用户 ID 或其他身份信息。
- 禁止提交用户目录、绝对文件路径、访问令牌、调试会话信息或包含这些内容的日志。
- 禁止提交未经脱敏的 Codex、Studio 或桌面截图；视觉验收图片必须只显示虚构示例数据，并在提交前人工复核。
- 预览和测试只能使用当前公开仓库名，或明显虚构、与真实本机环境无关的示例名称。
- 不得为了提高预览真实性而从运行中的 Codex 同步项目、任务、账号或团队数据。

## 预览数据要求

侧边栏预览使用 `src/renderer/src/preview-home.ts` 中的固定示例快照。修改该快照时必须同步更新测试中的完整允许列表，确保评审能够识别新增名称。布局测试必须继续覆盖虚构团队标识，禁止改成本机账号或团队名称。

## 提交前检查

1. 检查 `git diff` 中的项目、任务、账号、团队、邮箱和本机路径。
2. 检查新增图片及日志，确认没有真实环境信息。
3. 运行 `npm test -- --run tests/preview-layout.test.ts`，确认示例数据允许列表未被绕过。

违反以上任一规则时不得提交、发布或打包。
